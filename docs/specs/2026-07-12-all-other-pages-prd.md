# 第二部分：OK 页定点修复

---

## 8. calendar · 经营日历

**核心问题**：未来几天备货建议是前端 if-else 硬编码规则，和参谋的主动推送一样属于「伪智能」。

### 修复

| # | 改动 | 优先级 | 工作量 |
|---|------|--------|--------|
| 1 | **备货建议改为 API 驱动**：调用 `GET /advice/daily` 返回的 `recommendations` 或新增 `/env/seasonal` 接口的 `suggestions` 字段，不再用前端 if-else 拼接文案 | P0 | 0.3d |
| 2 | 加「一键生成采购清单」按钮：将当天策略中的商品建议加入采购清单 | P1 | 0.2d |
| 3 | 下拉刷新 | P0 | 0.1d |

**说明**：`/env/seasonal` 接口已存在（之前 calendar 修复中新增），但其 `suggestions` 字段是后端查表生成的静态数据。应扩展该接口使其结合商户历史数据生成动态建议，或直接复用 `/advice/daily` 的建议。
