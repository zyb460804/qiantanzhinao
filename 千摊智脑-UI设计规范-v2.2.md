# 千摊智脑 · UI 设计规范 v2.2（市井烟火 × 生鲜元气）

> 在 v2.1 地基上叠加 **主动智能感知层 / 丝滑转场 + 时段皮肤 / 成品质感 / 三个新功能模块**，并打通 **跨屏联动**。
> 视觉主张、令牌、字体、间距、无障碍基线沿用 v2.1（得意黑 + Space Grotesk + Noto Sans SC，OKLCH 生成，4pt 间距，WCAG AA）。本文只记录 **v2.2 变更与新增系统**。

---

## 一、主动智能感知层（小智从"被动问答"→"主动洞察"）

**核心理念**：AI 不该等用户来问，而该在用户打开App时就替他想一步。两条落地路径：

### 1. 经营台「小智今日洞察」卡
- 位置：Hero 下方、快览三连上方；暖金→叶绿渐变底，小智头像带 `live` 呼吸点（番茄红脉冲光环）。
- 内容：2 条可操作洞察（如"西红柿剩 2.4 斤 · 建议降 0.5 元清货"、"空心菜午后旺 · 建议补 3 斤"）。
- 每条带**「采纳」按钮**（改价 / 加入采购），点击即跨屏写入，按钮变 "已改价 / 已加入" 灰态。

### 2. AI 参谋「主动推送」面板
- 参谋页顶部独立面板，列出小智主动发现的机会点，同样支持一键采纳。
- 与经营台洞察共享 `adopt(nm, note, qt, kind)` 逻辑，两处采纳状态互相同步（同品名按钮统一置灰）。

### 3. 采纳 → 跨屏联动（关键）
- `kind='buy'`：把条目 push 进全局 `state.purchases`，**采购清单自动重渲**，Toast "已加入采购单 · XX"。
- `kind='price'`：触发 `addRevenue` 乐观更新，Toast "XX 已改价，损耗更省"。
- 全局状态 `state.revenue / state.collected / state.purchases` 贯穿所有屏，任何改动即时反映到经营台数字与采购清单。

## 二、丝滑转场 + 时段皮肤

### 1. 共享元素转场（View Transitions API）
- 所有 `show(name)` 导航包在 `document.startViewTransition()` 内（不支持或 `prefers-reduced-motion` 时优雅降级为直切）。
- 自定义 `::view-transition-new(root)` 缓出（`vtNew`：上浮 10px + 0.99 缩放 + 淡入，0.42s `ease-expo`），整屏切换如丝般连贯。
- 降级链：`canVT = !!(document.startViewTransition) && !reduce`。

### 2. 时段皮肤（早市 / 午市 / 晚市）
- 用 `:root[data-skin="dawn|day|dusk"]` 覆盖三枚氛围令牌：
  - `--glow-1` / `--glow-2`：控制 `body` 背景的双角径向光晕。
  - `--skin-tint`：控制时段分段控件选中态底色。
- 语义：
  - **早市 dawn**：清新晨光，微冷薄荷（cyan-green 偏冷）。
  - **午市 day**：标准生鲜绿（与品牌主色一致，默认）。
  - **晚市 dusk**：暮色暖金（gold/amber 偏暖，像收摊灯）。
- 自动判定：按 `new Date().getHours()` 选皮肤（<11 早市 / <17 午市 / 否则晚市）；`localStorage` 记忆用户手动覆盖。
- 经营台新增**分段控件**（🌅早市 / ☀️午市 / 🌆晚市），点击即切氛围 + Toast 反馈，全 App 背景、控件、光晕平滑过渡（body `transition: background .6s`）。
- 深色模式对各皮肤光晕单独微调（更暗、更收敛）。

## 三、成品质感打磨

### 1. 骨架屏（Skeleton）
- 7 个主屏（经营台/报告/库存/日历/同行榜/收款台/参谋）首次进入时，JS 注入 `.skel-wrap`（含 tall/row/short 占位），`shimmer` 流光动画 520ms 后移除，**全量 `reduce` 降级为直接显示**。
- 用 `skelSeen` 记忆已加载屏，避免每次切 Tab 都闪骨架（只在"第一次看"时加载叙事）。
- 与 View Transition 共存：骨架作为新快照的一部分出现，过渡自然。

### 2. 乐观更新（Optimistic UI）
- **语音记账完成**：`sheetDone` → 经营台数字 +9（模拟这笔卖出），Toast "记账已入账"，无需等后端回执。
- **收款台收款**：扫码/预设金额 → 立即把金额并入 `state.revenue` 与 `state.collected`，经营台「今日收款」与收款台大数字同步跳动。

### 3. 空状态（IP 小智陪伴）
- 采购清单全部勾选 → 小智摇摆庆祝空状态"采购清空啦！" + "重新编辑"。
- 小智 `.mascot` 自带 `wave` 摆姿态动画（transform-origin 底部）。

## 四、新增功能模块（真实可用原型）

### 1. 收款台 `screen-cashier`
- 大数字"今日已收（含记账）"，居中扫码大钮（叶绿渐变 + `scan-ring` 扫描脉冲）。
- **模拟收款**：点扫码钮随机抽 ¥6.5/12/23.5/35；或点预设金额 chip。
- 成功后展现 `pay-ok` 卡片：金额 + **语音播报文案**"🔊 微信收款 XX 元，已记入今日营业额"（呼应摊主"听得到才安心"的诉求）。
- "完成收款" → 回经营台并结算 Toast。

### 2. 经营日历 / 时令 `screen-calendar`
- 顶部**星期条**（一~日，今日高亮绿底）。
- **时令卡**：演示 7/11 落在"小暑"（7/7–7/22 消暑旺季），给出三行可执行建议：
  - 🛒 备货：西瓜/莲藕/绿豆芽/丝瓜；叶菜易蔫，少进勤补。
  - 📣 营销："消暑三件套"西瓜+绿豆+凉粉，满 20 送薄荷。
  - ⏰ 时段：11–13 家庭采买、17–19 下班高峰，提前码货。
- 下方 3 张"接下来几天"日卡（周日家庭日 / 周一机关食堂 / 周二平销），各有备货策略。

### 3. 同行激励榜 `screen-leaderboard`
- **本周附近摊主经营分排行**（6 行）：排名金/银/铜色（玉米金 / 冷蓝 / 暖铜），头像取摊名首字渐变圆。
- **本摊高亮**："王记蔬菜摊"标绿底 `me` 态，排第 3，制造"再努一把进前三"的目标感。
- 底部"冲榜任务"卡：把损耗率压到 2.5% 以下 → 经营分 +5，呼应经营台"损耗率 3.1%"。

## 七、无障碍（延续 v2.1 并加强）

- 全量 `prefers-reduced-motion`：跳过 View Transition、骨架屏、count-up、声波纹、扫描脉冲，直接给最终态。
- 触控目标 ≥44px（采纳钮、分段控件、预设金额、FAB 均达标）。
- 时段皮肤**只改变氛围光晕与强调色相**，**不降低正文对比度**；所有文字仍守 WCAG AA（正文约 14:1，远超标）。
- `aria-label` 保留在主题键、语音 FAB 等关键控件。
