## 二、发现并修复的真实 Bug

### Bug 1：边缘端 `pending_count()` 使用 SQLAlchemy 的 `scalar()` 方法

**位置**：`edge/main.py:94`

**问题**：`sqlite3.Cursor` 没有 `.scalar()` 方法，运行 `EdgeController.pending_count()` 会抛 `AttributeError`。

**修复**：改为 `fetchone()[0]`。

```python
# 修复前
n = conn.execute("SELECT COUNT(*) FROM pending_records WHERE synced=0").scalar()

# 修复后
row = conn.execute("SELECT COUNT(*) FROM pending_records WHERE synced=0").fetchone()
return int(row[0]) if row else 0
```

### Bug 2：`HX711Sensor` 初始化参数与配置不匹配

**位置**：`edge/main.py:138` 调用 `HX711Sensor(**config.get("hx711", {}))`，其中 `edge_config.json` 包含 `known_weight_g`。

**问题**：`HX711Sensor.__init__()` 原签名只有 `dout_pin`、`pd_sck_pin`、`simulate`，接收 `known_weight_g` 会抛 `TypeError`。

**修复**：在 `edge/weighing/hx711.py` 中让 `__init__` 接受 `known_weight_g` 并在初始化时自动去皮+标定。

```python
def __init__(
    self,
    dout_pin: int = 5,
    pd_sck_pin: int = 6,
    known_weight_g: float | None = None,
    simulate: bool | None = None,
):
    ...
    if known_weight_g:
        self.tare(times=5)
        self.calibrate(known_weight_grams=known_weight_g, times=5)
```

| 事项 | 阻塞原因 | 需要你做什么 |
