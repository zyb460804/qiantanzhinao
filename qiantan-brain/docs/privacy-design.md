# 经验云隐私设计 (Privacy Design)

千摊经验云聚合**跨商户匿名知识**（如「气温 >28°C 时西瓜销量 +35%」），用于增强单店建议。隐私是设计前提，不是事后补丁。

## 1. 已实现（当前版本）

| 机制 | 说明 | 代码位置 |
|------|------|----------|
| **最小样本阈值** | 聚合结果仅在 ≥ `MIN_MERCHANT_SAMPLE = 3` 家商户数据时才发布，否则回落行业规则 | `experience_cloud.py` |
| **仅统计聚合** | 服务端只算均值/计数，**原始个体数据不出库** | 三个聚合查询 |
| **不返回商户 ID** | 接口只返回商品、系数、分桶后的商户数 | `_apply_privacy` |
| **商户数分桶** | 精确计数改为 `<3 / 3-5 / 5-10 / 10-20 / 20+`，防止通过稀有计数重识别 | `_bucket_merchants` |
| **差分隐私噪声** | 数值聚合结果叠加 **Laplace 噪声**（预算 `PRIVACY_EPSILON`，默认 1.0），输出含 `privacy.epsilon` 与 `mechanism` | `_laplace_noise` / `_dp_value` |
| **查询预算闸** | 单 key 查询次数上限 `PRIVACY_QUERY_BUDGET`（默认 100），约束总信息泄露 | `_check_budget` |
| **聚合日志** | 发布洞察时只记录商品与分桶，不记个体 | `logger.info` |

配置项：`PRIVACY_EPSILON`（越小越私密）、`PRIVACY_QUERY_BUDGET`（环境变量）。

## 2. 设计要点

- **Laplace 机制**：对数值 `v` 叠加 `Laplace(0, sensitivity/ε)` 噪声。
  - `sensitivity` 按指标设定（天气影响 `5.0`、销量均值/总量 `2.0`）。
  - 理论上满足 **ε-差分隐私**：单条记录的增删对输出的影响受 `ε` 约束。
- **分桶 + 阈值** 共同构成「k-匿名」式防护，避免小样本被反推。
- **回落规则** 在样本不足或查询异常时返回文献/行业经验，保证可用性同时不泄露个体。

## 3. 规划中（尚未实现，答辩须如实表述）

- **联邦学习（Federated Learning）**：各商户本地训练、仅上传梯度，进一步避免原始数据集中。
- **敏感度自动估计** 与 **组合预算会计（Composition）**：多查询叠加时的总 `ε` 核算。
- **查询审计与告警**：异常高频查询识别。

> 答辩表述建议：「当前实现基于**最小样本阈值的匿名统计聚合 + Laplace 差分隐私噪声**，预留联邦学习扩展能力。」——**不要**直接表述为「已实现联邦学习」。
