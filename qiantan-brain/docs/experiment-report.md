# 实验报告（结构模板）(Experiment Report)

> 状态：**待实地采集**。以下为实验设计与指标模板，供 3–5 家商户试用后填充。

## 1. 实地试用设计

每家商户至少记录：

| 指标 | 说明 |
|------|------|
| 使用前库存记录方式 | 纸笔 / 其他软件 |
| 日均损耗率 | （损耗量 / 进货量） |
| 记账耗时 | 单次记账平均分钟 |
| 建议采纳率 | 采纳建议数 / 建议总数 |
| 建议准确率 | 商户主观评分或后续销量印证 |
| 用户满意度 | 1–5 分 |
| 连续使用天数 | 黏性 |

目标样本：**3–5 家** 生鲜/菜摊商户，连续使用 ≥ 2 周。

## 2. 算法对比实验（消融）

| 实验组 | 预测方法 | 变量 |
|--------|----------|------|
| Baseline A | 昨日销量 | — |
| Baseline B | 7 日移动平均 | — |
| Model C | 7/30 日移动平均 | — |
| Model D | 移动平均 + 天气 | +温度/降雨 |
| Model E | Prophet | 时序 + 节假日/周末 |
| Model F | Prophet + 天气 + 节假日 | 全变量 |

### 评价指标

- **精度**：MAE、RMSE、MAPE
- **业务**：损耗率、缺货率、建议采纳率

### 核心消融（证明「环境感知」有效）

```text
无环境变量
  vs 加入天气
  vs 加入天气 + 节假日
```

对比 MAPE / 损耗率 / 缺货率，量化环境因子的贡献。

## 3. 预期结论落点

- 环境感知（温度 / 降雨 / 节假日）在生鲜场景下应显著降低损耗率与缺货率；
- Prophet 在**数据量 > 30 天**后优于移动平均；冷启动（< 14 天）以规则/移动平均更稳，并设自动回退。
