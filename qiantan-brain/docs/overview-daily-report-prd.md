# 经营日报主动触达 PRD — 产出概览

## 做了什么
为「千摊智脑」撰写了**「经营日报主动触达」功能规格文档（PRD v0.1）**，定位为 MVP（可落地、可开发）。核心是把原本"需摊主主动打开 AI 参谋才能看到"的每日经营建议，升级为：
- **首页日报卡片**（打开即见，零权限依赖）
- **关键事件触发式订阅消息**（库存告急 / 暴雨预警 / 临期时点一次授权即可推一次）

## 关键决策
- **锚定真实痛点**：系统已有 `adoption_rate`（建议采纳率）基线约 **0.65**，摊主不主动打开 App 导致建议无人看 → 损耗/缺货。PRD 把"提升采纳率"定为北极星指标（→ 0.80）。
- **诚实处理微信平台约束**：普通商户工具类小程序**拿不到"每日定时无差别推送"的长期订阅资质**。因此明确将"每日定时推送"写入 **Non-Goals (NG1)**，不画饼；可行替代方案写入 P0/P2。
- **复用现有接口**：`/advice/daily`、`/inventory/alerts`、`/env/forecast`、`/behavior/feedback` 均已存在，MVP 主要新增首页卡片组件 + 订阅消息封装 + 后端事件判定服务。

## 文档结构（符合 feature-spec 模板）
问题陈述 → 目标(G1–G5) → 非目标(NG1–NG5) → 用户故事(7 条) → 需求(P0×5 / P1×3 / P2×4) → 成功指标(北极星+驱动+健康+滞后) → 开放问题 → 时间线依赖 → 风险合规 → 触达机制示意。
